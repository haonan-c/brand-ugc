# 提示词与数据合同

## 单一真源

- 模型阶段先输出 JSON。
- 使用 `schemas/*.schema.json` 校验 JSON。
- 通过校验后再渲染现有 Markdown/TXT 文件。
- 下游只读取 JSON，不从 Markdown 反向解析业务字段。

## 十二分镜

- `shots` 必须恰好为 12 项，`index` 连续为 1–12。
- 时间轴从 0 秒开始，按顺序连续推进，不重叠、不倒序。
- 对标视频镜头数不等于 12 时，只能合并或拆分已有内容。
- 视频解析的镜头必须标记 `observed` 或 `derived`。

## 语言

- JSON 键和枚举使用 Schema 定义的英文。
- 用户内容、说明、图片提示词和视频提示词使用中文。
- 不生成一套重复的完整英文提示词。

## 事实

- 产品事实只来自用户资料和产品图中的直接可见内容。
- 缺失信息使用中性表达，并写入 `needs_confirmation`。
- 不编造认证、销量、专利、评价、促销和功效。

## 媒体与日志

- 原视频不发送到远程服务。
- 视频分析仅发送本地最高 720p 代理和单声道音轨。
- 原视频仅用于本地十二帧抽取。
- 产品图、人物图和流程生成图可按阶段需要发送到 EvoLink。
- 原始响应日志仅保留请求/任务 ID、模型、状态、耗时、用量、模型输出和
  本地输入文件名；删除 Base64、密钥、上传凭据和临时 URL。

## 请求预算

- 四个业务结构化阶段各一次，Schema 失败各最多修复一次。
- 两个图片阶段各生成一张，不批量生成候选。
- 两个图片阶段各执行视觉 QA，失败各最多纠错生成一次并复检。
- 单次运行最多使用配置中的 14 次模型业务请求；文件上传、余额查询和任务
  状态轮询不计入模型业务请求。
